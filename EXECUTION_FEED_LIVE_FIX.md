# Live Execution Feed Fix

## Problem
The execution feed on the dashboard was not displaying trades in real-time. Trades would only appear after a page refresh because the chart was being updated on a 500ms interval (`interval-slow`) rather than in response to real-time WebSocket messages.

## Root Cause
The execution chart was using a **server-side callback** that triggered every 500ms:
```python
app.callback(
    Output('execution-chart', 'extendData'),
    Input('interval-slow', 'n_intervals')
)(update_execution_chart_data)
```

This meant:
1. Trades only checked on the interval timer (500ms max latency)
2. If page was not refreshed, the state tracking could get out of sync
3. Multiple trades in the same interval would only show the latest batch

## Solution
Converted the execution chart to use a **client-side callback** triggered by WebSocket messages, just like the OFI and metrics charts:

### Changes Made

#### 1. Updated `dash_clientside.js` (root)
Added a new clientside function that extracts the trades data from WebSocket messages:
```javascript
update_execution_chart: function (msg) {
    if (!msg || !msg.data) return window.dash_clientside.no_update;
    try {
        const data = JSON.parse(msg.data);
        if (data.type === "metrics_update" && data.trades) {
            return data.trades;
        }
    } catch (e) { console.error(e); }
    return window.dash_clientside.no_update;
}
```

#### 2. Updated `src/app/assets/dash_clientside.js` 
Added the same function to the assets version (mirrored file).

#### 3. Updated `src/app/dash_callbacks.py`
Replaced the server-side callback with a client-side callback:
```python
# ========== EXECUTION CHART STREAMING (Client-side for real-time) ==========
app.clientside_callback(
    ClientsideFunction(
        namespace='ws_clientside',
        function_name='update_execution_chart'
    ),
    Output('execution-chart', 'extendData'),
    Input('ws', 'message')
)
```

## Benefits
- **Real-time updates**: Trades appear immediately as WebSocket messages arrive
- **No page refresh needed**: Chart state stays in sync
- **Lower latency**: Client-side processing eliminates round-trip delay
- **Consistent architecture**: Now uses the same pattern as other streaming components
- **Network efficient**: Trades are already included in the WebSocket payload that's sent to all clients

## Technical Details
The WebSocket message structure already included trade data:
```json
{
  "type": "metrics_update",
  "trades": [extendData format],
  ...
}
```

The `trades` field contains the properly formatted `extendData` structure that Plotly requires:
- `{'x': x_data, 'y': y_data}` - the data points to append
- Indices for which traces to update (0=price, 1=buys, 2=sells)

This data is generated by the existing `update_execution_chart_data()` function on the server side (still used for initialization), then passed through to clients via WebSocket.

## Testing
All existing unit tests pass (209 tests), confirming no regressions were introduced.
